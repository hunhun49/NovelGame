# [기획서 07] AI 기반 사운드 및 오디오 오케스트레이션

## 1. 시스템 목표
사운드는 비주얼 노벨의 체감 품질을 크게 좌우합니다. 본 프로젝트에서는 AI가 직접 음악을 생성하지 않고, 등록된 오디오 자산 중에서 현재 장면에 맞는 요소를 선택하도록 설계합니다.

---

## 2. 오디오 레이어 구조

### 2.1 BGM Layer
- 메인 분위기 음악
- 장면 전환 시 크로스페이드

### 2.2 Ambience Layer
- 빗소리, 밤벌레 소리, 도시 소음
- 배경과 시간대를 강화

### 2.3 SFX Layer
- 문 여는 소리
- 발걸음
- 충격음
- 심장 박동

### 2.4 Voice Layer (Optional)
- 향후 TTS 또는 보이스 팩 확장용
- V1 필수 범위는 아님

---

## 3. 오디오 자산 메타데이터

### 3.1 BGM
- `bgm_id`
- `mood_tags`
- `scene_tags`
- `rating`

### 3.2 Ambience
- `ambience_id`
- `location_tags`
- `weather_tags`
- `time_tags`

### 3.3 SFX
- `sfx_id`
- `trigger_tags`
- `intensity`

---

## 4. AI 출력 데이터 예시
```json
{
  "audio": {
    "bgm_id": "night_tension_01",
    "ambience_id": "rooftop_wind_night",
    "sfx_id": null,
    "volume_profile": "quiet_tense"
  }
}
```

초기 버전에서는 `bgm_id`와 `sfx_id` 정도만 있어도 충분합니다.

---

## 5. 오디오 전환 규칙

### 5.1 BGM 전환
- 같은 계열이면 페이드만 적용
- 분위기가 크게 바뀌면 크로스페이드

### 5.2 SFX 재생
- 일회성 이벤트에만 사용
- 과도한 남발 금지

### 5.3 Silence도 연출로 사용
- 고백 직전
- 공포 장면 직전
- 폭력 직후

이처럼 `무음`도 하나의 의도된 상태로 취급해야 합니다.

---

## 6. 성인용/고어 장면 고려
성인용 혹은 강한 폭력 장면에서는 다음 원칙이 필요합니다.

- 오디오도 `rating`에 맞는 자산만 사용
- 일반판에서 제거된 자산은 후보군에서 제외
- 플레이어 설정에 따라 BGM은 유지하되 특정 SFX만 끌 수 있게 함

예:
- 고어 효과음 비활성화
- 성인 장면 전용 BGM 비활성화

---

## 7. Godot 구현 방향
- `AudioManager`가 모든 레이어를 관리
- 각 레이어별 `AudioStreamPlayer` 분리
- 전환은 코드 또는 `Tween`으로 제어

장점:
- 한 레이어 교체 시 다른 레이어에 영향이 적음
- 음량 프리셋 적용이 쉬움

---

## 8. 구현 아이디어
- V1은 `BGM + SFX` 2층만 먼저 구현해도 충분합니다.
- 이후 `Ambience`, `Voice`, `Duck/Burst`를 붙이면 됩니다.
- 사운드 시스템도 이미지처럼 **AI가 전체 라이브러리를 보지 않고, 엔진이 후보를 필터링한 뒤 선택하게 하는 구조**가 바람직합니다.
