# [기획서 04] 지능형 서사 생성 및 장기 메모리 관리 시스템

## 1. 시스템 목표
실시간 생성형 VN의 가장 큰 문제는 **정합성 붕괴**입니다.

예:
- 이미 고백한 관계를 다시 낯선 사람처럼 다룸
- 밤 장면인데 다음 문장에서 갑자기 아침이 됨
- 이미 부상한 캐릭터가 멀쩡하게 행동함

따라서 메모리 시스템은 단순 로그 저장이 아니라, **서사 정합성을 유지하기 위한 계층형 상태 시스템**이어야 합니다.

---

## 2. 기억 계층 설계

### 2.1 Short-term Memory
- 최근 6~12턴 원문
- 현재 감정 흐름
- 직전 장면 행동

용도:
- 자연스러운 대화 이어가기
- 억양과 분위기 유지

### 2.2 Scene Memory
- 현재 씬에서 중요한 사건
- 사용 중인 장소와 감정 상태
- 최근 자산 출력 상태

용도:
- 씬 단위 정합성 유지
- 배경/스프라이트 전환 논리 연결

### 2.3 Episode Memory
- 챕터 내 핵심 사건 요약
- 중요한 선택 결과
- 잠금 해제된 CG 이벤트 플래그

### 2.4 Canon Memory
- 세계관 핵심 설정
- 캐릭터 과거사
- 관계 축
- 영구 플래그

---

## 3. 어떤 정보를 저장해야 하는가
매 턴 종료 시 최소한 아래 정보는 구조화해서 남겨야 합니다.

```json
{
  "turn_id": 48,
  "player_input": "오늘은 네 얘기를 좀 더 듣고 싶어.",
  "ai_output_summary": "옥상에서 아리가 감정을 열기 시작함",
  "location": "school_rooftop_night",
  "visible_characters": ["airi"],
  "relationship_state": {
    "airi": 62
  },
  "flags": [
    "airi_confession_started"
  ],
  "content_rating": "mature"
}
```

원문 전체는 별도 로그로 저장하고, 프롬프트 재주입에는 요약과 핵심 상태만 사용합니다.

---

## 4. 서사 전개 제어

### 4.1 Pacing Control
엔진은 현재 서사의 속도를 느슨함/보통/고조로 단순 분류할 수 있어야 합니다.

활용 예:
- 너무 오래 일상 대화만 반복되면 사건 발생 플래그 제안
- 감정 고조 상태에서는 짧고 강한 대사 유도

### 4.2 Continuity Check
새 출력이 기존 설정과 충돌하는지 간단히 검사합니다.

초기 버전 아이디어:
- 위치 충돌
- 시간대 충돌
- 등장 캐릭터 충돌
- 이미 종료된 이벤트 재호출

이 검사는 별도 모델이 아니어도, 규칙 기반 검증만으로도 상당 부분 잡을 수 있습니다.

---

## 5. 요약 전략
모든 턴을 그대로 넣으면 컨텍스트 비용이 커지므로, 턴 누적 방식 대신 씬/에피소드 단위 요약을 사용합니다.

추천 방식:
- 매 턴: 짧은 로그 저장
- 씬 종료 시: scene summary 생성
- 챕터 종료 시: episode summary 생성

장점:
- 프롬프트 길이 안정화
- 로딩 속도 개선
- 세이브 파일 크기 제어

---

## 6. 분기 관리
이 프로젝트는 고정 스크립트 분기 대신 **상태 기반 분기**에 가깝습니다.

따라서 분기는 다음 조합으로 판단합니다.
- 관계 수치
- 플래그
- 이전 선택의 톤
- 잠금 해제된 자산
- 현재 rating lane

예:
- 고백 CG는 `trust >= 60` 과 `airi_confession_started`가 모두 필요
- 성인 이벤트는 `adult_enabled`와 특정 플래그가 모두 필요

---

## 7. 세이브와 메모리의 관계
세이브 시스템은 메모리의 결과물을 단순 복사하는 것이 아니라, **현재 서사를 재개 가능한 상태 패키지**여야 합니다.

포함 요소:
- 대화 로그
- 요약 정보
- 관계도
- 플래그
- 현재 장면 출력 상태
- 사용 중인 백엔드 설정 버전

이렇게 해야 저장 후 재개했을 때 말투나 장면 흐름이 급변하지 않습니다.

---

## 8. 구현 아이디어
- V1은 `요약 JSON + 최근 대화 + 플래그`만으로 시작합니다.
- 나중에 `정합성 검사기`, `사건 추천기`, `세계관 검색기`를 붙여 확장할 수 있습니다.
- 기억 시스템은 복잡해 보이지만, 첫 버전은 **최근 로그 + 씬 요약 + 캐논 설정**의 3층 구조면 충분합니다.
