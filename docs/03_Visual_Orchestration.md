# [기획 03] 자산 선택과 시각 출력 오케스트레이션

## 1. 목표

이 프로젝트의 이미지는 AI 생성이 아니라 `사용자 제공 자산 선택` 방식이다.  
즉, 백엔드는 텍스트와 연출 지시를 만들고, 실제 이미지는 `AssetLibrary` 가 검증한 자산만 사용한다.

현재 버전의 핵심 원칙:

- 이미지 생성 없음
- 외부 폴더 기반 자산 라이브러리
- `backgrounds`, `sprites`, `cgs` 세 카테고리만 지원
- 모든 출력은 `layered` 또는 `cg` 로 제한

## 2. 자산 카테고리

### 2.1 Background

일반 배경 이미지.

필수 필드:

- `id`
- `file`
- `location`
- `time_of_day`
- `weather`
- `rating`

선택 필드:

- `mood_tags`

### 2.2 Sprite

캐릭터 스탠딩 이미지.

필수 필드:

- `id`
- `file`
- `character_id`
- `expression`
- `pose`
- `outfit`
- `rating`

선택 필드:

- `state_tags`

### 2.3 CG

특정 이벤트 컷 일러스트.

필수 필드:

- `id`
- `file`
- `event_type`
- `character_ids`
- `rating`
- `required_flags`
- `blocked_flags`

선택 필드:

- `mood_tags`

## 3. 외부 manifest 구조

루트 파일은 반드시 `<library root>/manifest.json` 이어야 한다.

```json
{
  "version": "1",
  "backgrounds": [],
  "sprites": [],
  "cgs": []
}
```

상대 경로는 라이브러리 루트 기준으로 해석된다.

## 4. 현재 검증 정책

`AssetLibrary` 는 라이브러리를 로드할 때 다음을 검사한다.

- `manifest.json` 존재 여부
- 루트가 JSON object 인지
- `version` 존재 여부
- 각 항목의 필수 필드 존재 여부
- 중복 ID 여부
- 파일 존재 여부
- 지원하지 않는 `rating` 여부

현재 지원하는 rating:

- `general`
- `mature`
- `adult`
- `extreme`

검증에 실패하면 라이브러리 상태는 `invalid` 가 되고, 메뉴와 설정 패널에서 오류 메시지를 표시한다.

## 5. 후보군 전달 방식

AI 에게 전체 라이브러리를 보내지 않는다.  
대신 현재 상태에 맞는 후보군만 전달한다.

### 5.1 배경 후보

- 현재 `location_id` 와 유사한 배경 우선
- 현재 `rating_lane` 을 넘지 않는 배경만

### 5.2 스프라이트 후보

- 현재 활성 캐릭터와 관련된 스프라이트 우선
- 현재 `rating_lane` 을 넘지 않는 스프라이트만

### 5.3 CG 후보

- 현재 플래그 조건을 만족하는 CG 만
- 현재 `rating_lane` 을 넘지 않는 CG 만

이 구조의 목적:

- 존재하지 않는 자산 ID 생성 방지
- 현재 문맥과 동떨어진 자산 사용 방지
- 백엔드 프롬프트 토큰 절약

## 6. 출력 모드

### 6.1 Layered

- 배경 표시
- `left`, `center`, `right` 슬롯의 스프라이트 표시
- CG 숨김

일상 대화, 탐색, 관계 빌드업에 사용한다.

### 6.2 CG

- CG 표시
- 캐릭터 슬롯 숨김
- 배경은 유지하되 시각적으로 뒤로 물릴 수 있음

고백, 전환점, 성인 이벤트, 결정적 순간에 사용한다.

## 7. fallback 규칙

현재 구현은 자산 문제를 치명 오류로 처리하지 않고 연출적으로 복구한다.

- 배경 오류: 이전 배경 유지
- 스프라이트 오류: 해당 슬롯 숨김 또는 neutral/default fallback 사용
- CG 오류: `layered` 로 강등
- rating 초과 자산: 사용 불가
- flag 조건 미충족 CG: `layered` 로 강등

이 fallback 메시지는 상태 스트립에도 표시된다.

## 8. 현재 구현 한계

이번 라운드에서 의도적으로 제외한 것:

- 인게임 메타데이터 편집기
- 드래그 앤 드롭 자산 등록 도구
- 자산 자동 태깅
- 복수 CG 레이어 합성
- 애니메이션 스프라이트

즉, 자산은 외부에서 준비하고 게임은 읽고 검증하고 선택만 한다.

## 9. 운영 방향

실제 제작 단계에서는 자산 라이브러리를 다음처럼 나누는 것이 좋다.

- 일반판용 라이브러리
- 성인판용 라이브러리
- 테스트용 경량 라이브러리

특히 지금처럼 백엔드가 아직 안정화되지 않은 단계에서는 `테스트용 경량 라이브러리` 가 중요하다.  
작은 수의 배경, 스프라이트, CG 로 먼저 흐름을 검증해야 한다.
