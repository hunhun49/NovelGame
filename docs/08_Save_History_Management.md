# [기획서 08] 타임라인 기반 세이브 및 이력 관리 시스템

## 1. 시스템 개요
AI 게임의 특성상 '과거의 선택'이 중요하므로, 단순 스탯 저장을 넘어 대화의 맥락(Context) 전체를 스냅샷 형태로 관리합니다.

## 2. 스냅샷 데이터 구조
세이브 시점의 다음 정보를 패키징하여 DB(Supabase)에 저장합니다.
- **Conversation Log:** 현재까지 진행된 전체 대화 텍스트 리스트.
- **World State:** 캐릭터 호감도, 유저 스탯, 현재 위치(Background ID).
- **AI Context:** Gemini가 기억해야 할 핵심 요약본(Summary).

## 3. 주요 기능
- **Auto-Save:** 매 대화 턴이 종료될 때마다 로컬 스토리지에 임시 저장.
- **Timeline Rollback:** 유저가 이전 분기로 되돌아가고 싶을 때, 해당 시점의 스냅샷을 불러와 AI 프롬프트를 재구성.
- **Multi-Slot:** 유저당 최소 3개 이상의 수동 세이브 슬롯 제공.
