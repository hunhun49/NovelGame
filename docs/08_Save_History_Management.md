# [기획서 08] 타임라인 기반 세이브 및 이력 관리 시스템

## 1. 시스템 개요
생성형 비주얼 노벨에서 세이브는 단순한 위치 저장이 아니라, **현재 서사를 다시 이어서 생성할 수 있는 상태 패키지**여야 합니다.

즉 세이브 파일은 다음 세 가지를 함께 담아야 합니다.
- 플레이 기록
- 현재 세계 상태
- AI 재개에 필요한 요약 정보

---

## 2. 저장 단위

### 2.1 Auto Save
- 매 턴 종료 후 저장
- 최근 몇 개만 순환 보관

### 2.2 Manual Save
- 플레이어가 원하는 시점 저장
- 슬롯 이름과 메모 입력 가능

### 2.3 Quick Save / Quick Load
- 데스크톱 VN 문법을 따르는 빠른 저장

### 2.4 Timeline Node
- 특정 분기점 또는 주요 이벤트 지점 표시

## 4. 왜 대화 로그 전체가 필요한가
요약만 있으면 재개는 가능하지만, 생성형 게임에서는 직전 뉘앙스가 중요합니다.

따라서 권장 방향:
- 전체 로그 저장
- 프롬프트에는 최근 로그 일부 + 요약 사용

이 구조면 저장은 풍부하게, 런타임은 가볍게 가져갈 수 있습니다.

---

## 5. 타임라인 롤백
이 프로젝트의 핵심 기능 중 하나는 과거 분기점으로 돌아가 다른 서사를 보는 것입니다.

이를 위해 세이브에는 다음 정보가 중요합니다.
- 당시 플래그 상태
- 관계 수치
- 현재 장면 상태
- 장기 요약
- 잠금 해제된 이벤트 여부

롤백 후에는 이후 대화 로그를 폐기하고, 해당 시점부터 새로운 분기를 생성합니다.

---

## 6. 유저 자산과 세이브의 관계
이 프로젝트는 유저 자산 라이브러리를 사용하므로, 세이브는 자산 파일 자체를 복사하지 않고 **자산 ID 참조**만 저장하는 편이 현실적입니다.

주의점:
- 자산이 삭제되면 예전 세이브가 완전 재현되지 않을 수 있음
- 따라서 자산 메타데이터 변경 시 경고가 필요함

보완 아이디어:
- 세이브 시 자산 라이브러리 버전 해시 저장
- 누락 자산 발견 시 폴백 규칙 적용

---

## 7. 성인용 콘텐츠 관련 저장 옵션
데스크톱 환경에서는 플레이어 프라이버시가 중요합니다.

추천 옵션:
- 최근 세이브 썸네일 비활성화
- 성인 장면 세이브 마스킹
- 특정 슬롯 숨김
- 로그 파일 저장 여부 선택

---

## 8. 손상 복구 전략
- JSON 파싱 실패 시 이전 오토세이브 복구 제안
- 필수 필드 누락 시 최소 재개 모드 제공
- 자산 누락 시 장면만 단순화해 복원

---

## 9. 구현 아이디어
- V1은 JSON 기반 슬롯 시스템으로 충분합니다.
- 나중에 세이브가 많아지면 슬롯 인덱스만 따로 관리해도 됩니다.
- 중요한 것은 저장 속도보다 **언제든 믿고 되돌릴 수 있다는 체감**입니다.
