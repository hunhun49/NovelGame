# 자산 라이브러리와 오디오
## Summary
이미지와 오디오는 Godot 프로젝트 내부 `res://`가 아니라 외부 라이브러리 루트의 `manifest.json`을 기준으로 읽는다. 자산 선택은 백엔드가 아니라 클라이언트 `AssetLibrary`가 다시 검증하고 캐시한다.

## Key Points
- `AssetLibrary.reload_library()`는 루트 경로에서 `manifest.json`을 읽고 backgrounds, sprites, cgs, bgms, sfxs를 모두 인덱싱한다.
- 각 자산은 rating lane 검증을 거치며, CG는 `required_flags`와 `blocked_flags`까지 확인한다.
- 턴 요청용 후보 묶음은 현재 위치, 활성 캐릭터, 현재 플래그를 기반으로 구성된다. 즉 AI는 전체 라이브러리가 아니라 상황에 맞는 부분 집합만 받는다.
- 스프라이트 해상은 `sprite_id` 우선이지만, 실패하면 캐릭터의 감정 이미지나 썸네일, 마지막에는 neutral/default 스프라이트 fallback까지 내려간다.
- `AudioManager`는 두 개의 BGM 플레이어를 번갈아 써서 crossfade를 만들고, UI 버튼 hover/down/up과 대사 타이핑 SFX도 공통 처리한다.
- 현재 데모 매니페스트는 배경 2개, 스프라이트 2개, CG 1개, BGM 2개, SFX 4개 정도의 최소 샘플 구성을 가진다.
