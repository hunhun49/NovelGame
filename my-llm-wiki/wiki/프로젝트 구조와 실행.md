# 프로젝트 구조와 실행
## Summary
저장소는 Godot 클라이언트, FastAPI 백엔드, 그리고 위키용 문서 저장소로 나뉜다. 실제 게임 실행의 중심은 `godot-novel-maker/project.godot`이고, AI 턴 생성은 별도 `backend/` 서버가 맡는다.

## Key Points
- `godot-novel-maker/`는 게임 씬, UI 스크립트, autoload 서비스, 데모 자산을 포함한 실제 플레이어 클라이언트다.
- `backend/`는 `/health`, `/v1/backend/prewarm`, `/v1/story/turn`을 제공하는 최소 FastAPI 서버이며 Ollama를 직접 호출한다.
- `my-llm-wiki/`는 프로젝트 문서와 그래프 산출물 저장소다. 게임 실행에는 직접 관여하지 않는다.
- 개발 시작점은 보통 `backend` 서버를 띄우고, Godot에서 `project.godot`를 열어 설정 패널로 자산 루트와 백엔드 URL을 맞추는 흐름이다.
- 백엔드가 없어도 `SettingsManager.apply_demo_session()`과 stub 모드로 UI/상태 흐름을 검증할 수 있다.
