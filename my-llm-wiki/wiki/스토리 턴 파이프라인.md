# 스토리 턴 파이프라인
## Summary
플레이어 입력 한 턴은 `NovelScene -> NarrativeDirector -> AiClient -> FastAPI backend -> GameState 반영 -> 렌더링` 순서로 흐른다. 시각 연출과 오디오는 모두 응답 후 `NarrativeDirector`에서 최종 확정된다.

## Key Points
- `NarrativeDirector._build_turn_request()`는 `persona`, `world`, `runtime_state`, `session`, `recent_conversation`, `asset_candidates`를 묶어 백엔드에 보낸다.
- 자산 후보는 `AssetLibrary.build_candidate_bundle()`에서 생성되지만, 턴 요청에는 배경 3개, 스프라이트 4개 등 작은 상한으로 잘라서 보낸다.
- `AiClient`는 backend health와 warm 상태를 추적하고, 필요하면 `/v1/backend/prewarm`을 먼저 호출한 뒤 턴 요청을 재개한다.
- 서버 응답 후 `NarrativeDirector`는 배경/CG/스프라이트/BGM/SFX를 rating lane에 맞게 다시 검증한다. 실패하면 fallback 메시지를 쌓고 기존 상태를 유지한다.
- 주 대사 캐릭터는 `_ensure_primary_dialogue_character_visible()`로 강제로 중앙 슬롯에 보정된다. 즉 AI가 잘못된 슬롯을 돌려줘도 UI 측에서 다시 정렬한다.
- 관계도 변화, 플래그, 최근 대화 로그, 요약 후보, 롤백 스냅샷까지 모두 `GameState`에 기록된다.
